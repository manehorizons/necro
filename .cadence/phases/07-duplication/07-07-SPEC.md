# 07-07 — Duplication detector

> Source of truth: [docs/necro-design-spec.md](../../../docs/necro-design-spec.md).
> This slice = build-order step 9 (§8): duplication. §4 method —
> "AST-normalize (strip names/whitespace), hash subtrees, find collisions
> (token-based)". Implemented on the **existing phase-05 tree-sitter parser**
> (no jscpd dependency). Completes the deterministic detector set; surfaced as a
> `Duplication` section in `necro scan`.

## Objective

Detect copy-paste clones — including renamed copies — by tokenizing each file
through tree-sitter, normalizing identifiers and literals away, and finding
matching token sequences across and within files, surfaced as a `Duplication`
section in `necro scan`. Local, free, deterministic.

## Acceptance Criteria

### AC-1: Normalized token stream
Given a TypeScript source file
When it is tokenized
Then it yields a stream of normalized tokens where identifiers collapse to `ID` and literals to `LIT` (Type-2), keywords/operators/punctuation keep their kind, and comments are dropped — and only the tokenizer names tree-sitter/TS constructs (core invariant §3).

### AC-2: Cross-file clone detection
Given two files that share a normalized token sequence of at least `minTokens`
When duplication runs
Then a clone group is reported listing each location (`file:startLine-endLine`) and the matched token length; a sequence that appears only once is not reported.

### AC-3: Type-2 (renamed) clones
Given two blocks that are identical except for renamed identifiers and changed literal values
When duplication runs
Then they are detected as a clone (normalization erases the names), while structurally different code of the same length is not.

### AC-4: Threshold and within-file clones
Given the `minTokens` threshold (default 50, configurable)
When duplication runs
Then sequences shorter than `minTokens` are never reported, and a block duplicated **within a single file** is detected as a clone (two locations in the same file).

### AC-5: Maximal, de-duplicated clones
Given a clone longer than `minTokens`
When it is reported
Then it appears once as a single maximal region per location — not as a swarm of overlapping sub-window matches.

### AC-6: Surfacing in scan
Given a project containing duplicated code
When `necro scan` runs
Then it prints a `Duplication` section (one entry per clone group, worst-first by token length) and `--json` includes a `duplication` array; a project with no clones shows no section; `necro fix` is unaffected.

## Constraints

- **Reuse the phase-05 tree-sitter parser** (`getParser`) to tokenize — **no jscpd / no new parser dependency**, consistent with the "single distributable" goal.
- **Normalization is Type-2:** identifiers → `ID`, literals → `LIT`, comments dropped; keywords/operators/punctuation keyed by their token kind (whitespace-insensitive by construction).
- **Core invariant §3:** tree-sitter/TS token kinds appear only in the tokenizer; the clone-finding algorithm operates on the normalized token stream alone.
- **Algorithm:** rolling-hash over `minTokens`-length windows → collision groups verified by token equality → greedy extension to the maximal match → mark covered windows so a long clone is reported once (AC-5). Cross-file and within-file.
- **Surfacing:** a `Duplication` section in `necro scan` + a `--json` `duplication` array. `ScanResult` gains `duplication`; it rides the heavy (tree-sitter) axis, so `necro fix` (passing `complexity: false`) skips it.
- **Threshold:** `minTokens` default 50, overridable via a `duplication` config block.
- Report-only; **no LLM** (locked #3); no auto-fix.
- Out of scope: cross-language clones, fuzzy/Type-3 (gapped) clones, clone-based refactor suggestions (LLM, step 13).

## Decisions (resolved open questions)

- **Approach:** token-hash on tree-sitter (reuse phase-05 parser), not jscpd — lean, no heavy dependency.
- **Clone type:** normalized **Type-2** (renamed copies match), per the §4 "strip names" method.
- **Surfacing:** new `necro scan` section + `--json` `duplication` (multi-axis consistency).
- **Threshold:** `minTokens` default 50, configurable.
