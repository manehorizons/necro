# Design ‚Äî `necro bench` + Accuracy page (rec-006 v1)

**Date:** 2026-06-11
**Rec:** `rec-20260610-006` ‚Äî Public measured-accuracy benchmark ("weaponize the eval harness")
**Status:** approved design, ready for CADENCE draft

## Goal

Externalize necro's already-internal eval harness into a **published credibility
artifact**: an Accuracy docs page backed by measured precision/recall (dead-code
triage) and pass-rate (duplication refactor), regenerated by a thin, reproducible
`npm run bench` runner that writes a provenance-stamped snapshot.

Positioning: **"the dead-code tool with measured accuracy."**

## Decisions (settled ‚Äî do not relitigate)

1. **Primary deliverable = the Accuracy page** (credibility artifact). The runner
   exists to keep the page's numbers honest and reproducible, not as a headline
   end-user feature.
2. **necro-only numbers in v1.** No knip / ts-prune head-to-head yet ‚Äî that is a
   clean fast-follow once a defensible fairness protocol is documented.
3. **Numbers come from a thin bench runner ‚Üí committed snapshot.** Not hand-typed
   on the page; not generated at docs-build time.
4. **Both corpora, triage headlined.** Lead with triage precision/recall as the
   "measured accuracy" headline; duplication pass-rate is a clearly-labeled second
   section with a *different* (structural) metric.
5. **Runner surface = `npm run bench` (repo-internal), NOT a published `necro bench`
   CLI subcommand.** Rationale: `package.json` ships `"files": ["dist"]` only ‚Äî the
   corpus (`test/fixtures/`) is **not** in the npm tarball, so a published command
   would have no corpus to run against. "Reproducible by anyone" honestly means
   "anyone with the repo checkout," which is the correct reproduction model for a
   benchmark (you need the corpus to reproduce).

## What already exists (reused, not rebuilt)

- `src/triage/eval.ts` ‚Äî `runEval(cases, client, opts) ‚Üí EvalMetrics`
  (`{ total, truePositives, falsePositives, falseNegatives, precision, recall,
  rows, breakdown }`; positive class = "dead"). `loadEvalCases(path)`.
- `src/refactor/eval.ts` ‚Äî `runRefactorEval(cases, client, opts) ‚Üí RefactorEvalMetrics`
  (`{ passRate, rows }`), structural scoring (`COLLAPSE_RATIO = 0.5`). `loadEvalCases`.
- Corpora (already in-repo, already provenance-documented via `SOURCES.md`):
  - **Triage:** `test/fixtures/triage-realrepo/cases.json` ‚Äî 48 cases (33 alive /
    15 dead) across `honojs/hono@61d6d66d` (19) + `trpc/trpc@c7360d4e` (29).
  - **Dup:** `test/fixtures/refactor-dup-realrepo/cases.json` ‚Äî 12 cases across
    `trpc/trpc@c7360d4e` (4) + `drizzle-team/drizzle-orm@48e54060` (8).
- Live evals already run via `*.live.test.ts` (model in the loop, gated, never CI).

Bench is **orchestration + provenance + snapshot**, not new scoring.

## Architecture

Three independently-testable units.

### 1. `src/bench/snapshot.ts` ‚Äî schema, provenance, (de)serialize

The result contract the runner writes and the page reads.

```ts
interface BenchCorpusResult {
  id: "triage" | "dup";
  metricKind: "precision-recall" | "pass-rate";
  sources: { repo: string; sha: string; cases: number }[];
  n: number;
  metrics:
    | { precision: number; recall: number; f1: number;
        truePositives: number; falsePositives: number; falseNegatives: number }
    | { passRate: number; passed: number; total: number };
}

interface BenchResults {
  schemaVersion: 1;
  methodologyVersion: 1;
  generatedAt: string;        // ISO; injected (not Date.now in pure code)
  necroVersion: string;       // from src/version.ts (build-time-inlined)
  model: string;              // model id used for the run
  corpora: BenchCorpusResult[];
}
```

- `f1` derived from precision/recall.
- Functions: `summarizeTriage(EvalMetrics, sources) ‚Üí BenchCorpusResult`,
  `summarizeDup(RefactorEvalMetrics, sources) ‚Üí BenchCorpusResult`,
  `serialize(BenchResults) ‚Üí string` (stable key order, trailing newline),
  `parse(string) ‚Üí BenchResults`.
- **No I/O, no clock, no model calls here** ‚Äî pure, fully unit-testable.

### 2. `src/bench/run.ts` ‚Äî orchestrator

```ts
async function runBench(opts: {
  corpus: "triage" | "dup" | "all";
  triageClient: TriageClient;     // injected
  refactorClient: RefactorClient; // injected
  now: string;                    // injected ISO timestamp
  model: string;
}): Promise<BenchResults>
```

Loads the selected corpora, calls `runEval` / `runRefactorEval`, maps results
through `snapshot.ts` summarizers, stamps provenance. Clients and clock are
**injected** ‚Üí unit-testable with stubs, zero live calls in `npm test`.

### 3. `src/bench/cli-bench.ts` ‚Äî runner entry (`npm run bench`)

Tiny CLI: parses `--corpus triage|dup|all` (default `all`), `--out <path>`
(default `bench/results.json`), `--dry-run` (print, don't write). Constructs the
**real** live clients (same model wiring as the `*.live.test.ts` evals; requires
`ANTHROPIC_API_KEY`), stamps the real timestamp + `model` + `necroVersion`, calls
`runBench`, writes the snapshot via `serialize`.

`package.json`: add `"bench": "tsx src/bench/cli-bench.ts"` (or vitest-runner
equivalent if `tsx` isn't already a dep ‚Äî confirm during planning). **Not** added
to `src/cli.ts` / the published `necro` binary.

## Data flow

```
npm run bench  ‚Üí  cli-bench (live clients, real clock)  ‚Üí  runBench()
                    ‚îú‚îÄ runEval(triage cases)        ‚Üí EvalMetrics
                    ‚îî‚îÄ runRefactorEval(dup cases)   ‚Üí RefactorEvalMetrics
                  ‚Üí BenchResults (+ provenance)  ‚Üí serialize ‚Üí write bench/results.json (committed)
                                                                        ‚îÇ
website/ Accuracy page  ‚îÄ‚îÄimports committed bench/results.json at build time‚îÄ‚îÄ‚îò
                                                                        ‚Üí headline + tables + captions
```

Docs build **never** calls the model. The published numbers are always the
committed snapshot. Regeneration is a deliberate maintainer step.

## Accuracy page ‚Äî `website/src/content/docs/guide/accuracy.md`

Renders from the committed snapshot (Astro: import `bench/results.json`, or a
small content loader ‚Äî resolve exact mechanism in planning against the site's
existing content collections).

- **Headline:** triage **precision / recall / F1** on the 48-case real-repo
  dead-code corpus. Caption pulled live from snapshot fields:
  *"necro `vX` ¬∑ model `‚Ä¶` ¬∑ corpus hono@61d6d66d + trpc@c7360d4e ¬∑ measured `<date>` ¬∑ N=48."*
- **Second section:** duplication-refactor **pass-rate** (12 cases,
  trpc@c7360d4e + drizzle@48e54060), explicitly framed as a different structural
  metric (clone-collapse), **not** precision.
- **Methodology:** corpus selection, location-weighted LLM judging (triage),
  structural `COLLAPSE_RATIO` scoring (dup), and the **non-determinism caveat**
  (model in loop ‚Üí a measured snapshot, not a guarantee). Include a
  "Reproduce it yourself: `npm run bench`" block (needs repo + `ANTHROPIC_API_KEY`).
- **Limitations:** sample size, repo selection, no competitor comparison yet.

Add the page to the docs nav/sidebar.

## Honesty guardrails (the whole point)

- Every number on the page traces to a snapshot field ‚Äî no hand-typed metrics.
- The caption renders `model` + corpus SHAs + `generatedAt` + `N` from JSON, so
  stale numbers are *visibly* stale.
- Methodology states metric definitions and limitations plainly.

## Testing

- **Unit (in `npm test`, deterministic, no live calls):**
  - `snapshot.ts`: serialize/parse round-trip; `f1` derivation; provenance
    completeness (all required fields present).
  - `run.ts`: `runBench` with **stub** clients asserts it wires both corpora and
    shapes `BenchResults` correctly (counts, metricKind, sources).
- **Schema-guard test:** assert every field the Accuracy page reads exists in the
  committed `bench/results.json` (catches page ‚Üî snapshot drift).
- **Live regeneration stays manual** (like existing `*.live.test.ts`) ‚Äî never CI.

## Out of scope (v1) ‚Äî explicit, all clean follow-ons

- knip / ts-prune head-to-head comparison + fairness protocol.
- A published `necro bench` end-user CLI subcommand.
- Bundling the corpus into the npm package.
- New corpus cases / additional repos.

## Acceptance criteria (for the CADENCE draft)

- **AC-1:** `npm run bench` produces a provenance-stamped `bench/results.json`
  covering both corpora (triage precision/recall/F1 + dup pass-rate), with
  `necroVersion`, `model`, `generatedAt`, per-corpus `sources`/`n`.
- **AC-2:** An Accuracy page renders the headline triage metrics and the dup
  pass-rate **from the committed snapshot**, with a provenance caption and a
  methodology + limitations + reproduce section, and is reachable from the docs nav.
- **AC-3:** Deterministic unit tests cover `snapshot.ts` and `runBench` (stubbed
  clients), plus a schema-guard test tying the page's read-fields to the snapshot;
  full suite stays green and the docs site builds.
